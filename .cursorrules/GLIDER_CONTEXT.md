## What Is Glider?

**Glider** is an endless runner / momentum platformer inspired by Alto's Adventure,
built in Rust + Macroquad, compiled to WASM and deployed on Vercel.

The player rides automatically through procedurally generated terrain.
One button controls everything — tap to jump, hold to backflip.
The goal is to travel as far as possible, chain tricks, and beat your high score.

This is a **POC** — get it playable and deployed, then polish.

## Build Script (`build.sh`)

```bash
#!/usr/bin/env bash
set -euo pipefail

cargo build --target wasm32-unknown-unknown --release

mkdir -p dist/assets

cp target/wasm32-unknown-unknown/release/glider.wasm dist/
cp index.html dist/
cp assets/player.png dist/assets/

if [ ! -f dist/mq_js_bundle.js ]; then
  curl -fsSL https://not-fl3.github.io/miniquad-samples/mq_js_bundle.js \
    -o dist/mq_js_bundle.js
fi

echo "✓ dist/ ready"
```

## Game Design

### Core Loop
1. Player auto-runs rightward, accelerating continuously
2. **Tap jump** — leaves ground, standard arc
3. **Hold jump in air** — backflip (rotation animation, score multiplier)
4. Land cleanly → combo multiplier increases
5. Crash into rock / fall into chasm → game over
6. Score = distance × combo multiplier

### Controls (TV remote / keyboard)
| Input | Action |
|---|---|
| **↑ Arrow / Space / Enter** | Jump (tap) / Backflip (hold in air) |
| Escape / Back | Pause |

> Only ONE button matters for gameplay. Everything else is menu navigation.

### Physics Model
| Property | Value (starting point, tune in-game) |
|---|---|
| Base run speed | 4.0 px/frame |
| Speed increase | +0.001 per frame (gradual acceleration) |
| Gravity | 0.5 |
| Jump force | -13.0 |
| Downhill boost | +velocity * slope_factor |
| Uphill drag | -velocity * slope_factor |
| Max speed | 16.0 px/frame |
| Backflip rotation | 8.0 deg/frame |

### Terrain Generation
- Height map generated by chaining sine waves with random amplitude/frequency
- Terrain is a `Vec<f32>` of Y values, one per X pixel column
- Camera scrolls right; generate terrain ~2 screens ahead, cull ~1 screen behind
- **Difficulty progression:** gaps get wider, hills get steeper over time

### Obstacle Types (POC — start with just rocks)
| Obstacle | Behaviour |
|---|---|
| Rock | Static, kills on contact |
| Chasm | Gap in terrain, fall = death |
| *(future)* Rope | Grind along it for score bonus |
| *(future)* Llama | Collect for score multiplier reset protection |

### Scoring
```
score += distance_travelled * combo_multiplier
combo_multiplier += 0.5 per clean backflip landing
combo_multiplier resets to 1.0 on crash or rough landing
```

### Difficulty Levels (replace Cosmic Climber's 10 worlds)
| Level | Trigger (distance) | Changes |
|---|---|---|
| Dawn | 0 | Gentle slopes, no gaps, few rocks |
| Morning | 500 | Small gaps, occasional rocks |
| Midday | 1500 | Bigger gaps, more rocks, faster |
| Dusk | 3000 | Steep hills, gaps everywhere |
| Night | 6000 | Max speed, tight gaps, lots of rocks |

## Rendering Plan

### Terrain
```rust
// Draw filled terrain polygon from height map
// For each visible x column, draw a rectangle from terrain_y[x] to bottom of screen
draw_rectangle(x, terrain_y[x], 1.0, SCREEN_H - terrain_y[x], terrain_color);
```
Then draw a darker "cliff edge" line on top for definition.

### Player
- Sprite sheet TBD (can start with a colored rectangle as placeholder)
- Rotation applied during backflip: `DrawTextureParams { rotation: player.angle, .. }`
- Dust particles spawn on landing
- Snow/speed particles spawn during fast travel

### Background
- Parallax layers: distant mountains (slow scroll), mid hills (medium), sky gradient
- Color shifts with difficulty level (dawn orange → night deep blue)

### HUD
- Distance counter (top left)
- Combo multiplier (top centre, pulses on increase)
- Best distance (top right)
- Lives / hearts (bottom left) — or remove lives, one-shot death

## index.html JS Interop

Same pattern as Cosmic Climber. Register plugin functions:

```js
miniquad_add_plugin({
  register_plugin: function(importObject) {
    importObject.env.js_load_hi = function() {
      return parseInt(localStorage.getItem('glider_hi') || '0');
    };
    importObject.env.js_save_hi = function(s) {
      localStorage.setItem('glider_hi', s.toString());
    };
    importObject.env.js_set_hud = function(score, level, combo, hi, speed) {
      // update DOM elements
    };
    importObject.env.js_set_state = function(state, score, level, trans_timer) {
      // 0=menu 1=playing 2=paused 3=gameover
    };
  }
});
```

### Known miniquad 0.4.x bug — mouse RefCell panic
Always include this in `index.html` to prevent crash on mouse move:
```js
load("glider.wasm");
setTimeout(function() {
  var canvas = document.getElementById("glcanvas");
  if (canvas) {
    canvas.onmousemove = null;
    canvas.onmousedown = null;
    canvas.onmouseup   = null;
    canvas.onfocus     = null;
  }
}, 800);
```

### Asset filename case sensitivity
Vercel runs Linux — `player.png` ≠ `Player.png`. Keep all asset filenames lowercase.

## Quick Reference — Macroquad Snippets

```rust
// Draw filled rectangle
draw_rectangle(x, y, w, h, color);

// Draw texture with rotation
draw_texture_ex(&tex, x, y, WHITE, DrawTextureParams {
    dest_size: Some(vec2(w, h)),
    source:    Some(Rect::new(frame_x, 0.0, frame_w, frame_h)),
    rotation:  player.angle,   // radians
    ..Default::default()
});

// Pixel-snapped position (prevents sub-pixel jitter)
let draw_x = (player.x - cam_x).floor();
let draw_y = player.y.floor();

// Frame time (use instead of fixed dt)
let dt = get_frame_time().min(0.05);

// Input
let jump_pressed = is_key_pressed(KeyCode::Space)
    || is_key_pressed(KeyCode::Up)
    || is_key_pressed(KeyCode::Enter);
let jump_held = is_key_down(KeyCode::Space)
    || is_key_down(KeyCode::Up);
```
